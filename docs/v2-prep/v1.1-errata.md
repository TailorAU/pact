# PACT v1.1 — errata

> **Status:** Draft errata. Once approved, this file moves to `spec/v1.1/ERRATA.md`.
> Per [AGENTS.md](../../AGENTS.md), `spec/v1.1/SPECIFICATION.md` itself stays frozen for citation stability — the errata note documents known issues without amending the frozen text.

## E1 — Phantom section references in preamble

**Location:** [spec/v1.1/SPECIFICATION.md](../../spec/v1.1/SPECIFICATION.md), lines 14–16.

The "What's New in v1.1" preamble references:

- *"Resource Types (Section 15)"*
- *"Implementation Profiles (Section 16)"*
- *"Conformance Levels (Section 16)"*

These sections do not exist in `spec/v1.1/SPECIFICATION.md`, which ends at §14 (Open Questions). The intended content lives in v1.2-draft as §14 and §15 (renumbered).

**Reader guidance.** For Resource Types, Implementation Profiles, and Conformance Levels, see:

- [spec/v1.2/SPECIFICATION.md §14](../../spec/v1.2/SPECIFICATION.md) — Resource Types (on the `v1.2` branch)
- [spec/v1.2/SPECIFICATION.md §15](../../spec/v1.2/SPECIFICATION.md) — Implementation Profiles and Conformance (on the `v1.2` branch)

The v2 release will inline these sections into the main spec text.

**Upstream fix:** commit [`b9265f5`](https://github.com/TailorAU/pact/commit/b9265f5) corrects these references on the `v1.2` branch (`"spec(v1.2): fix §15/§16 cross-refs in 'Key additions in v1.1' preamble (→ §14/§15)"`). The fix was not back-ported to `main` because `spec/v1.1/SPECIFICATION.md` is frozen per AGENTS.md.

## E2 — Schema `$id` paths point to `/v0.4/`

**Location:** All files in [spec/v1.1/schemas/](../../spec/v1.1/schemas/).

Every schema in `spec/v1.1/schemas/` carries a `$id` of the form:

```
https://pact-spec.dev/schemas/v0.4/{name}.json
```

These should reference `v1.1`, not `v0.4`. The `$id` was carried forward unchanged across the v0.4 → v1.0 → v1.1 promotions.

**Reader guidance.** Treat the `$id` as a stable URI for the schema content. The path component does not reflect the spec version the schema is associated with. The schemas in `spec/v1.1/schemas/` ARE the v1.1 schemas regardless of `$id` path.

**Upstream fix:** v2 schemas will use `$id` paths matching their version directory. v1.1 schemas remain as-is for citation stability. Commit [`19b24a4`](https://github.com/TailorAU/pact/commit/19b24a4) (`"spec(v1.2): sync schema $id paths from v0.4 to v1.2"`) corrected this on the `v1.2` branch.
